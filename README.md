# Øvelsesplanlegging 110-sentral

Dette repoet samler arbeidet med å planlegge, gjennomføre og evaluere en spilløvelse for ett vaktlag på en fiktiv 110-sentral, B43, under kurs i Stavern.

## Deleside

- `index.html` er samlet dokumentportal for gruppen.
- GitHub Pages-lenke når publisering er aktivert: `https://rgrodem.github.io/brannskolen-110-ovelsesplanlegging/`
- GitHub-repo: `https://github.com/rgrodem/brannskolen-110-ovelsesplanlegging`

GitHub Pages kunne ikke aktiveres mens repoet er privat med nåværende GitHub-plan. Bruk derfor repo-lenken som delbar Git-side for de som får tilgang til repoet. README-en under fungerer som klikkbar dokumentoversikt i GitHub.

## Ramme

- Sted: Stavern
- Sentral: B43, fiktiv øvelsessentral
- Kontrollrom: 6 operatørplasser
- Øvende vaktlag: 4 personer, 3 operatører og 1 vaktleder
- Øvingsform: spilløvelse med motspill
- Action time: ca. 90 minutter
- Scenario: totalforsvarsrelevant hendelse, konkret scenario avklares etter målvalg

## Kursmål

Kursmål 2.2.12 Øvelsesplanlegging og evaluering:

- kjenne til veiledere og håndbøker for øvelsesplanlegging og systematisk evaluering
- planlegge og gjennomføre øvelser under veiledning
- planlegge og gjennomføre systematisk evaluering av øvelser og hendelser

## Arbeidsprinsipp

Vi følger kursets oppskrift og malene i mappen. Mål og evalueringskriterier skal være på plass før selve 90-minutters dreiebok skrives. Scenarioet skal teste målene, ikke motsatt.

Siden B43 er en fiktiv sentral og vaktlaget ikke normalt jobber sammen, skal øvelsen ikke primært evaluere lokal systembruk, LEO eller arbeidsplasspesifikke rutiner. Øvelsen skal heller gi læring i rollefordeling, situasjonsforståelse, samvirke og strukturert håndtering i et kunstig, men realistisk, kontrollromsmiljø.

5. Spillkort for motspillere

7. Evalueringsskjema med ODCR/AAR
8. Presentasjon for før-brief og ettergjennomgang

## Arbeidsdokumenter

- `Strategi_gjennomforing_ovelsesplanlegging.html` viser produksjonsløpet for 1,5 dag.
- `scenario_totalforsvar.md` samler scenarioalternativer og anbefalt retning før konkret dreiebok skrives.
- `plan_alternativ_b_totalforsvar.md` beskriver overordnet løsning for trafikkulykke med militær kolonne og sekundær brann.
- `realistisk_110_ovingsmodell.md` justerer scenarioet mot normal 110-arbeidsflyt: nødtelefon, utalarmering, fremkjøring, første ressurs fremme og driftsfase.
- `leveranseplan_realistisk_ovelse.md` beskriver hva den ferdige øvelsespakken faktisk skal inneholde: ressursplan, innringerplan, dreiebok, spillkort, bilder, sikkerhet, evaluering og presentasjon.
